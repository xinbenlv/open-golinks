## Overview

每个链接有 QR 码, 可定制 caption (含 CJK) + logo, 可下载 PNG. 移植 master `components/QrCodeEditor.vue` UI + `src/routes/qr.ts` 后端.

兼容性要求: master 已暴露 `/qr/:slug.png` 和 `/qr/d/:slug.png`, 这些 URL 可能已被用户保存或嵌入. v2-hono 可以新增 `/api/v1/qr/:slug`, 但必须保留 master 兼容路径.

## API 设计

### `GET /api/v1/qr/:slug?format=png&caption=...&logo=<url|none>`

```
// 200 image/png
// 404 if slug 不存在 / 软删
// 400 if caption 太长 (> 100 chars) 或 format 不支持
```

服务端 LRU 缓存 PNG (key = `${slug}:${urlHash}:${caption}:${logo}`, TTL 1h, 1000 entries).

### Master 兼容路径

```text
GET /qr/:slug.png?caption=...&addLogo=true
GET /qr/d/:slug.png?caption=...&addLogo=true
```

- `/qr/:slug.png` 返回 `image/png`
- `/qr/d/:slug.png` 返回 `image/png` + `Content-Disposition: attachment; filename="<slug>.png"`
- 参数名保留 master 的 `addLogo`; 新 UI 可内部用 `logo`, 但兼容路径必须接受 `addLogo`

## 风险

| 风险 | 缓解 |
|---|---|
| `@napi-rs/canvas` Bun 不兼容 | W0 spike 验证; fallback 方案 B (纯客户端 canvas) |
| NotoSansCJK 字体加载阻塞首次请求 | 启动时 preload + LRU 缓存 PNG |
| 字体文件入 git 让 repo 变大 ~20MB | 接受 (一次性), 或用 LFS / build-time download |
| QR 在小手机相机识别率低 | error correction level = H, 留白边距 |
