## Overview

标记为 `metadata.show_warning: true` 的链接, redirect 时不直接 302, 先 SSR 警告页给用户确认. 用 `?confirm=1` 绕过.

## Deliverables

新文件:
- `src/routes/warn.ts` — SSR `GET /warn/:slug`, 返回最小 HTML (不依赖 SPA bundle)
- `src/web/components/WarnToggle.tsx` — Edit 页里的 "Mark as warning" 开关
- `tests/e2e/F6-warn.test.ts`
- `tests/browser/F6.spec.ts`

修改:
- `src/routes/redirect.ts`: 命中 link 后检查 `metadata.show_warning` + `req.query.confirm !== '1'` → 302 到 `/warn/:slug`
- `src/web/pages/Edit.tsx`: 加 toggle, 写入 `metadata.show_warning`
- `src/server.ts`: 注册 `/warn/:slug` 路由 (在 redirect 之前, 因为 `warn` 在 redirect.ts 的 RESERVED set 里)

## 依赖与现状

- F2 (因为依赖 metadata 写入路径)
- DB: 用 `links.metadata.show_warning` (boolean); `RESERVED` set 已包含 "warn"
- env: 无新增
- SSR 性能目标: P99 < 50ms

## API 设计

`GET /warn/:slug` 返回 HTML (不是 JSON):

```html
<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="utf-8" />
    <title>Warning: External link</title>
    <style>/* inline minimal css */</style>
  </head>
  <body>
    <main class="warn">
      <h1>⚠️ 即将跳转外部链接</h1>
      <p>You are about to visit:</p>
      <code class="dest">{escape(link.url)}</code>
      <p class="meta">Short link: /{slug}</p>
      <div class="actions">
        <a href="/{slug}?confirm=1" class="btn-proceed">继续访问 Proceed</a>
        <a href="/" class="btn-cancel">取消 Cancel</a>
      </div>
    </main>
  </body>
</html>
```
